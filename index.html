📘 Hướng dẫn cấu hình Firebase (làm theo 5 bước bên dưới, mất khoảng 5-10 phút)

Firebase là dịch vụ lưu trữ dữ liệu online miễn phí của Google (gói Spark), đủ dùng cho quy mô một đơn vị công an phường/xã. Bạn cần tự tạo một dự án Firebase riêng (miễn phí, không cần thẻ tín dụng) để dữ liệu tin báo được lưu trữ dưới quyền quản lý của chính đơn vị mình.

1 Tạo dự án (Project) Firebase
  1. Truy cập console.firebase.google.com và đăng nhập bằng tài khoản Google.
  2. Nhấn "Add project" / "Thêm dự án".
  3. Đặt tên dự án, ví dụ: congan-tanbinh-tinbao (không dấu, không khoảng trắng).
  4. Ở bước Google Analytics, có thể chọn "Không kích hoạt" (không bắt buộc cho ứng dụng này).
  5. Nhấn "Create project" và chờ Firebase khởi tạo xong.
2 Tạo Cloud Firestore Database
  1. Trong menu bên trái, chọn Build → Firestore Database.
  2. Nhấn "Create database".
  3. Chọn vị trí máy chủ gần Việt Nam nhất: asia-southeast1 (Singapore) hoặc asia-southeast2 (Jakarta). Lưu ý: không thể đổi vị trí sau khi tạo.
  4. Chọn chế độ "Start in production mode" rồi nhấn "Create" (không chọn "test mode" vì mode này sẽ tự khoá quyền truy cập sau 30 ngày).
3 Bật đăng nhập Email/Password & tạo tài khoản Admin
  1. Trong menu bên trái, chọn Build → Authentication, nhấn "Get started" nếu là lần đầu.
  2. Vào tab "Sign-in method", chọn "Email/Password", bật công tắc Enable ở dòng đầu tiên, nhấn "Save".
  3. Chuyển sang tab "Users", nhấn "Add user".
  4. Nhập email và mật khẩu cho tài khoản admin duy nhất, ví dụ: admin@congantanbinh.local (email không cần dùng thật, chỉ dùng để đăng nhập). Nhấn "Add user".
💡 Đây chính là tài khoản dùng để đăng nhập vào ứng dụng ở bước 7. Nên tạo đúng 1 tài khoản và chỉ chia sẻ cho cán bộ được giao quản trị dữ liệu. Ứng dụng không có chức năng tự đăng ký — chỉ có thể thêm tài khoản mới qua Firebase Console như trên.
4 Thiết lập Firestore Security Rules (quyền đọc/ghi)

Vào Firestore Database → tab "Rules", xoá nội dung mặc định, dán đoạn dưới đây rồi nhấn "Publish". Đây là rule khuyến nghị — chỉ cho phép đọc/ghi khi đã đăng nhập (khớp với tài khoản admin vừa tạo ở bước 3):

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tinbao_cases/{caseId} {
      allow read, write: if request.auth != null;
    }
  }
}

Nếu muốn chạy thử nhanh trước khi tạo tài khoản admin, có thể tạm dùng rule mở dưới đây rồi quay lại thay bằng rule trên sau khi hoàn tất bước 3:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tinbao_cases/{caseId} {
      allow read, write: if request.time < timestamp.date(2027, 1, 1);
    }
  }
}
⚠️ Rule mở phía trên chỉ nên dùng tạm để test — bất kỳ ai có 6 thông số cấu hình đều đọc/ghi được dữ liệu, và tự khoá sau ngày 01/01/2027. Sau khi đã tạo tài khoản admin ở bước 3, hãy chuyển hẳn sang rule request.auth != null ở trên để bắt buộc đăng nhập mới truy cập được dữ liệu.
5 Đăng ký Web App để lấy 6 thông số cấu hình
  1. Vào Project settings (biểu tượng bánh răng ⚙ cạnh "Project Overview" ở góc trên bên trái) → tab "General".
  2. Kéo xuống mục "Your apps", nhấn biểu tượng Web ( </> ) để thêm ứng dụng web.
  3. Đặt tên hiển thị, ví dụ: tinbao-web. Không cần tick "Also set up Firebase Hosting".
  4. Nhấn "Register app" — Firebase sẽ hiện ra đoạn mã firebaseConfig chứa đúng 6 giá trị cần dùng:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "ten-du-an.firebaseapp.com",
  projectId: "ten-du-an",
  storageBucket: "ten-du-an.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:xxxxxxxx"
};

Nhấn "Continue to console" để hoàn tất (không cần cài npm/SDK gì thêm).

6 Nhập thông số vào ứng dụng này
  1. Sao chép từng giá trị trong firebaseConfig ở bước 5 vào đúng ô tương ứng bên dưới: apiKey → API Key, authDomain → Auth Domain, projectId → Project ID, storageBucket → Storage Bucket, messagingSenderId → Messaging Sender ID, appId → App ID.
  2. Ô "Tên Collection dữ liệu" giữ mặc định tinbao_cases (phải trùng với tên trong Rules ở bước 4 — nếu đổi tên collection thì sửa lại rule tương ứng).
  3. Nhấn "🔌 Lưu & Kết nối".
  4. Màn hình đăng nhập admin sẽ hiện ra ngay sau đó — xem tiếp bước 7.
7 Đăng nhập lần đầu
  1. Sau khi kết nối ở bước 6, một màn hình đăng nhập sẽ che toàn bộ ứng dụng.
  2. Nhập đúng email và mật khẩu của tài khoản admin đã tạo ở bước 3, nhấn "Đăng nhập".
  3. Đăng nhập thành công, badge phía trên chuyển thành 🟢 Đã đồng bộ Firebase và tên tài khoản admin hiện ở góc phải thanh menu, có nút "Đăng xuất".
  4. Nếu quên mật khẩu, nhấn "Quên mật khẩu?" để Firebase gửi email đặt lại (cần email admin là email thật, nhận được thư).
  5. Nếu chưa kịp tạo tài khoản admin, nhấn "Chưa có tài khoản admin? Vào mục Cấu hình" ở màn hình đăng nhập để quay lại chỉnh sửa mà không cần đăng nhập.
💡 Từ lúc đăng nhập thành công, dữ liệu nhập/sửa/xoá sẽ tự động đồng bộ lên Firebase theo thời gian thực, xem được từ nhiều máy tính khác nhau cùng lúc (đều cần đăng nhập cùng tài khoản admin). Vẫn nên dùng nút "Sao lưu dữ liệu" định kỳ trong mục "Danh sách vụ việc" để có thêm một bản sao lưu độc lập.

Cấu hình Firebase (lưu dữ liệu online, đồng bộ nhiều máy)

👆 Xem hướng dẫn chi tiết từng bước ở khối phía trên nếu chưa có sẵn thông số. Sau khi có đủ 6 giá trị, nhập vào form dưới đây.

Chưa kết nối Firebase — dữ liệu chỉ lưu tạm trong phiên làm việc.
Dữ liệu được lưu tạm trong phiên làm việc hiện tại — vào mục "⚙ Cấu hình hệ thống" để kết nối Firebase và lưu trữ online, hoặc dùng "Sao lưu dữ liệu" / "Phục hồi dữ liệu" để sao lưu thủ công.

Xuất báo cáo thống kê

Chọn khoảng thời gian tiếp nhận và loại vụ việc cần thống kê. Để trống "Từ ngày" / "Đến ngày" nếu muốn lấy toàn bộ thời gian.
